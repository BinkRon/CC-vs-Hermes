# Handoff 交接台账（运行态）

> **这是"跨 session 怎么接棒"的真相源。** 每个需求编号一个块，**只保留最新一棒**（新棒覆盖旧棒，旧棒靠 git 历史考古）。
> 规则见 `contract.md` 契约 8。需求清单与状态见 `backlog.csv`；整体进度看板见 `progress.md`。
>
> **领活铁律**（契约 8.1）：推进交付物的 session 开工前，先在 `backlog.csv` 选一个**`依赖`列全部 done 且 `状态=draft`（无人认领）**的需求绑定其 ID，**领取即把 `状态` 切出 draft（占位）**再动手；接**在制品**（状态非 draft/done）则只读该 ID 最新一棒的"下一步"续做、不重新领。做完/换手/搁置时三处同步：本文件 + `backlog.csv 状态` + `progress.md`（契约 8.5）。纯讨论/流程/元层 session 不绑不留棒。
> **并行**：`依赖`互不挡的需求可由不同 session 同时做（如 CH-06 与 CH-08）；有依赖的按序（CH-SYN2 等 CH-06/CH-08 全 done 才可领）。共享工作树时并入合订本那一步须串行（契约 8.6）。

## 一棒的格式

```
## {需求ID} {标题}
- **第 N 棒 · YYYY-MM-DD**
  - 完成摘要：≤3 个要点（可选）
  - 下一步：足够明确，让下一棒 1–2 分钟内无歧义接手（必填）
  - 阻塞项：需要人工/PM 动作的事项；无则写「无」
```
