## 各轴收尾结论
- **轴一「一个 Loop 的设计思路」**：**信任模型的程度，决定壳做多薄；是否拥有模型，决定能不能动第二根杠杆。** 控制侧、输出侧是**信任驱动的"改壳"**（越不敢信模型、壳越厚）；输入侧是**所有权驱动的"改模型"**（拥有模型才能把压缩形态往权重里训——意图，非本仓库可证；这是 Hermes 多出的第二根杠杆）。CC 押注一个可控托管模型 → 单杠杆；Hermes 面对异质开源模型群、又自己拥有模型 → 双杠杆。反直觉：做模型的人反而更不信任自己的模型。
- **轴二「Loop 之外的设计思路」**：**五种关系底下是同一个 master variable——「这个 agent 假设有没有人正盯着」。** CC 处处假设「有人在」（会回到会话、会点确认、当队长、变强要你签字），Hermes 处处假设「没人在」（自己保活、沉默≠同意、数据库协调、绕开你的会话变强）。而在场契约不是自由选择，是**所有权链条**逼出来的：谁拥有模型（轴一）→ 部署（04）→ 代码（05）→ 有没有厂商兜底 → 敢不敢假设有人在。反直觉：做「最小开源壳」的一方，反因没人替它在场，只能把「独立存在」做到最重。诚实边界：这根脊不是五等份（04/06/07 在场主导，05/08 实为所有权、在场只是侧面），且「对同类」的在场联系是解读非铁证、ACP 外来 agent 互通仍待查证。
