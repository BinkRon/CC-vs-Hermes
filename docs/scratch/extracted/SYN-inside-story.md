# 综合①：一个 Loop 是怎么设计的

*三侧拆完拼回来：一个回合的一生，与所有分叉的最终去向。*

> **【TL;DR / 核心洞察】**
> 读完三侧，最该带走的不是「两家有多不同」，而是两家的 Loop 八成是同一门手艺 ——机械循环同构、装配骨架同一张、协议地板互相通用。真正的分叉只有 8 处 ，而且几乎全部通向两个变量： 敢多信这个模型 （信任，决定壳的厚度）； 拥不拥有这个模型 （所有权，决定有没有第二根杠杆）。
> Claude Code 押注一个自家托管的强模型——通向两杠杆的 7 处分叉上，它的答案全是「不铺、不分、不训」，单杠杆且轻用； Hermes 面对一群参差的开源模型、又自己训模型——改壳、改模型两根杠杆一起拉。外加一处诚实的例外：有一处分叉谁的杠杆都不归，它只是「产品和底座长得不一样」。

## 三张解剖图都对，可你还是没见过这台机器跑起来

【问题框】
  前三章像三张解剖图：01 拆「循环为什么转得下去」，02 拆「每轮那个请求包怎么拼」，03 拆「吐出来的生料怎么加工」。每张图都对——可解剖图有个天生的缺陷： 看完三张，你还是没见过这台机器跑起来的样子 。更要紧的是，三章各自拆出的那些分叉散落在各处， 它们到底通向哪里、是不是同几个变量在反复作祟，光靠解剖图也拼不出来 。

所以这一章不新增任何机制，只干两件事：先把一个回合从头到尾走一遍（8 站，每站标清「这里两家一样」还是「这里分了叉」），让机器真的转起来；再把全部分叉收拢到一张图上，看它们最终汇向哪里。每一站、每处分叉，都回指三章里已过门的结论。

## 一个回合的一生：三张解剖图，拼回一条时间线

三章各沿一侧下刀，这一节换个方向：顺着 时间 走。从你敲下一句话开始，到 agent 把回答交还给你，一个回合恰好路过三章的全部领地。下面逐站走一圈，每站标出它属于哪章、两家在这一站是共用手艺还是分了叉。
先看灰色再看颜色：8 站里有 3 站（唤醒、调模型、生料回来） 一处分叉都没有 ——这就是「八成共享」的直观版；分叉挂在少数几站上，琥珀色的通向「改壳」，绿色的那颗全轴只有一颗、通向「改模型」，虚线的那颗谁都不通向。

## 8 处分叉的去向：6 处汇进改壳，1 处汇进改模型，1 处溜出去

时间线告诉你分叉「在哪」，这一节回答分叉「通向哪」。把 8 处分叉从站点上摘下来、按根因排队，去向惊人地整齐——但「整齐」里必须保住一个诚实的区分：汇进改壳的六处，三处是因为 不信 而加厚，三处是因为 异质 而适配。这不是同一件事。
0.5 秒该看到的：左边 8 处分叉，右边只有 两个实心出口＋一个虚线出口 ——六处流进第一个、一处流进第二个、一处溜走。每处分叉括注出处章节，数字都是原章过过门的。
一个 03 章警告过、这里必须重申的掰歪： 适配 ≠ 不信任 。方言归一、缓存分档、窗口缩放这三处分叉，就算配的全是强模型、只要 provider 杂就得铺——把它们也说成「Hermes 不信模型」，故事更顺，但不真。所以改壳杠杆内部诚实地分成两股。另有一处分叉（纪律看模型下菜）横跨两股——正是一群模型的存在，才让「模型行不行」变成一个需要应对的变量： 异质是土壤，不信是长在上面的作物 。
再看那颗唯一的绿色。六处改壳分叉说到底是「壳厚壳薄」的 程度差 ；训回权重那一处是 种类差 —— Claude Code 结构上就没有这个选项，它不拥有模型。这就是「第二根杠杆」的确切含义：不是 Hermes 更勤奋，是它的处境里多了一类可做的事。

## 两套系统各自咬合，拆件混装谁都装不上

把分叉归了因，还剩最后一层要说透：这些选择不是八个独立的取舍，是 两套互相咬合的系统 ——每家的每个选择，都以它其余的选择为前提。

[Claude Code] Claude Code —— 「一个可控托管模型」咬合出的一套
  - 前缀 真·统一 ——不必分叉，因为只有一个模型；
  - 敢用 固定阈值 ——窗口大小是已知数；
  - 壳对格式 不设防 ——模型受训可靠，喊错名只回报错让它自纠；
  - 防懒 交给模型自觉 ——相信「我做完了」的自述；
  - 回报是：判断权交给模型， 天花板随模型升级自动抬高 。

[Hermes] Hermes —— 「一群异质模型＋自有权重」咬合出的一套
  - 前缀 按模型分叉 ——Google 系、GPT 系各要各的话术；
  - 阈值 随窗口走 ——群里有小窗口也有大窗口；
  - 壳加 纠偏层 ——不能假设第三方模型把名字喊对；
  - 收工设 验证门 ——编码入口不放行任何模型（含自家）的自述；
  - 能训进权重的 训进去 ——第二根杠杆只有它拿得起。

用三个反事实试一下就知道「咬合」有多紧。 把 CC 的统一前缀搬进 Hermes？ 第一天就碎——Google 系要「别停」、GPT 系要「执行铁律」、阿里系连自己叫什么都要壳来纠正。前缀分叉不是设计品味，是模型群的物理现实。 把 Hermes 的方言归一层搬进 CC？ 无物可翻——它只有一个 API，那一层一行都用不上。壳厚不是勤奋，壳薄也不是懒惰，都是处境的倒影。
最有意思的是第三个反事实：给 CC 装 Hermes 的验证门？能装——而且正在装。停止钩子的口子一直留着（结构上与验证门同族），内部灰度里已经有「非平凡改动须独立验证代理裁决」的契约。这说明「锁定」分两种： 处境锁死的不会收敛 （异质、所有权——除非处境本身变了）， 纯信任偏好的正在收敛 （防懒这道门，两家方向一致、只差默认与否）。信任线是活的，处境线是死的。

【VERDICT】
  判词： 别问「谁做得对」，问「你的处境像谁」。这份对比的单位从来不是一张功能对照表，而是两套各自咬合的系统——单拆一件装进对方，多半既装不上、也没必要。

## 01 章留下的反直觉，现在可以讲完整了

01 章结尾留了一个反直觉：做模型的那家，反而更不信任模型说的「我做完了」。只看控制侧，这像一个悖论；把三章摊平到汇流图上，它拆成 两层来源不同的不信任 ——
一层是对「 任何模型的自述 」的不信，连自家也不例外：在编码入口，验证门对自家模型照拦不误（这道门按产品场景发放——聊天入口默认关，见 01 章；它不信的是「模型自述」这件事本身，不是无差别猜疑）。这一层来自专业——越清楚权重是怎么训出来的，越知道「我做完了」只是一个待核实的声明，不是结论。另一层是对「 别人家模型 」的经验性不信：纠偏层、名单加压，兜的是没被这样训练过的第三方模型，自家模型反倒不是主要服务对象。这一层来自生态——开源世界里没法假设可靠。
所以那句反直觉的完整版不是「做模型的人更悲观」，而是： 做模型的人，处境里的变量更多 。Anthropic 的处境里这两层都不存在——一个模型、自家调教、持续变强，押注它就好；Nous 的处境里两层都在，于是能训进权重的训进去，训不进去的用壳挡。

## 你的处境像谁，就抄谁的作业

若你基于一个强托管模型做 agent ：抄 Claude Code ——壳做薄、判断权交给模型，天花板随模型涨；但要接受模型犯浑时薄壳不兜底，并且记住它的「薄」只薄在上层，协议地板（回传重塑、配对补全）一点不能省。
若你要跑多种／开源／自托管模型 ：抄 Hermes ——那六处改壳分叉就是你的施工清单：三处加厚（验证门、按模型下菜、纠偏层）、三处适配（缓存分档、方言归一、窗口缩放）。至于第七处（训回权重），只在你也训模型时才对你开放。
一句话带走： 壳的厚度不是能力问题，是信任问题；第二根杠杆不是技术问题，是所有权问题。

## 悬而未决：六条别被两个变量收编的边界

【悬而未决 / 防夸大】
  - 「8 条」不是穷举所有差异，是「结构性分叉」的账： 记忆注入的「便签 vs 底稿」（02：CC 当会过期的便签、每轮召回叮嘱验证 vs Hermes 当不该被冲淡的底稿、冻进前缀当权威——02 章将其当切面处理、生命周期归轴二，本章沿用）、催办哲学（01）、图片存档顶替方式（03）、防死兜底的深浅（01）等程度差与切面差异留在原章、未计入分叉账。跨读原章看到「第 9 处不同」时，先问它是种类差还是程度差。
  - 例外真的只有一条吗？ 动态注入 44 vs 8 是三章里唯一被明确标为「另一回事」的分叉。两个变量解释力强，但别拿它收编一切——下一处分叉出现时，先问它汇入哪里，别先假设答案。
  - 「训回权重」的诚实边界沿 02 章： 离线管线存在、与运行时同源可证；线上权重是否真在这批数据上训练过，本仓库证不到。第二根杠杆是「拿得起」，不是「已建功」。
  - 信任线正在收敛： CC 的验证代理契约还锁在灰度特性门后。「防懒 0 道」是今天的默认，不是永远的哲学——本章拍的是快照，不是终局。
  - 「八成共享」有一部分不是英雄所见略同，是直接沿用 ——Hermes 回传规范化的注释点名沿用 CC 的做法。共享手艺的来源里有行业收敛，也有互相抄作业。
  - 时间线是单回合视角： 回合与回合之间——谁触发、谁保活、人跑了怎么办——是另一个世界的问题，那是轴二整条线（04–08 章）的主场。
