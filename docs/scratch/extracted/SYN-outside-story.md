`综合② · 轴二 Loop 之外是怎么设计的`
五种关系 · 一个总变量

# 它假设，有没有人正盯着？

五种关系看着是五件事—— 何时醒来、听谁改写、在哪动手、和谁协作、怎么变强 。但把它们并在一起，每一次分野都由同一个问题决定： 这个 agent，假设有没有人正盯着它？ Claude Code 处处假设「有人」， Hermes 处处假设「没人」。而这个假设不是品味选择——是所有权处境替它选好的。

## 半夜三点，没有人在屏幕前

## 同一个变量，五次落到同一侧

下面这张图是全章的骨。中间一条脊，是「假不假设有人在场」这个总变量；五种关系横跨其上——每一次， CC 都落向「有人」一侧， Hermes 都落向「没人」一侧。节点上的圆点，标的是 这一行到底有多「在场」 ：并非五等份，后两行真正在起作用的是另一条线（所有权），在场只是它的侧面。

## 在场契约，是所有权逼出来的

为什么 CC 敢假设「有人」，Hermes 只能假设「没人」？不是胆量差别，是 处境 差别。顺着"谁拥有什么"往下推，每一环都在把两家推向相反的默认——直到最后那个问题： 敢不敢假设有人会在。

## 先回答这一个问题，后面五件事就定了

## 这根脊，不是五等份

【悬而未决 / 防夸大】
  - 两条线，别硬拧成一条。 对时间／环境／同类，「在场」是正面主轴；对人／对自己，真正在分的是 所有权链 ，在场只是它的侧面（治理层有没有人盯／变强要不要你签字）。硬说成「在场的五次显形」就是以偏概全——所以本章把两条线分开画（脊上的空心点即标此）。
  - 「对同类」那一行是解读，不是铁证。 「人当队长 vs 数据库协调」是对机制的合理解读，非代码里可直接证的设计意图（半点标注即此）。
  - 「变强」这一行，两家都别信自己。 无论 CC 的「外包给下一代基座」还是 Hermes 的「握着训练杠杆」，画的都是 意图与潜力、不是已经做到 ——两家仓库 都证不到「你的使用让它权重级变强」 （CC 断在没这能力，Hermes 断在产数据管线没接你的活会话）。脊上 row5 标的是「谁有可能」，别读成「谁已经在越用越强」。
  - 五关系的边界仍有一处未闭。 「对同类」是否还应含 外来 agent 互通 （如 Hermes 的 ACP 适配器），07 章尚未覆盖、待查证；若那格材料成立，本综合「五关系」的完备性自评要回来再调——不因为故事顺就盖章。
