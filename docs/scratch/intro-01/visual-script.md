# INTRO-01 视觉脚本（design-principles Part 2.1 五问）

## 1. 一句话论点
这本册子把 harness 沿两条正交轴拆开——先解剖一个回合之内的四侧，再按补集穷尽回合之外的关系——每章带一把固定的提问仪器；导读只给地图与问法，不给任何结论。

## 2. 最大卡壳点
读者看到侧栏「轴一 · Loop 之内 / 轴二 · Loop 之外」不知所云：两轴各是什么、为什么这么切、凭什么说不漏。→ 重档给「两轴地图」。

## 3. 叙事顺序
问题（怎么比才不漏不偏）→ 对象与支点 → 拆法（两轴地图 + README 审计轶事）→ 读法（两把仪器）→ 章节地图 → 方法与边界。

## 4. 全序重量分配表
1. **两轴地图**（重档 · 全章唯一图 · 手工 HTML 非 mermaid——形状是"结构/地图"非流程；内圈实线=解剖、外圈虚线=补集三分，颜色只编码"内/外"这一维）
2. **章节地图行表**（中档 · 每章一行统领问题，label 与侧栏逐字咬合）
3. thesis TL;DR 绿框（开篇范式配件）
4. Hermes README 首句制品引文（轻 · 紫左规=Hermes 语义色，真实制品允许进盒）
5. `00·问题` 灰框（开篇范式配件）
6. **安静基线**：对象与支点 / 拆法正文 / 读法 / 方法与边界——全部裸文。

## 5. 重档草案
两轴地图：上半区实线框「轴一 · 一个回合之内」= 输入→计算→输出→控制四步横排循环（每步：侧名 + 一句职责 + 章号徽标；计算侧徽标=「不立章 · 散在各章」）+ 回环注一行；下半区虚线框「轴二 · 回合之外」= 补集三分三列（时空：04 时间/06 环境；其他主体：05 用户/07 其他 Agent；它自己：08）。读者 10 秒看到：内=解剖一个循环、外=不属于循环的三类去处；每格挂章号即目录。无两家对比→不适用差异高亮，近单色。

## 纪律（本章特有）
- 零剧透：章节地图只列问题；总变量/各章 verdict/完备性正式论证不出现。
- 轴二计数不押"五"：结构锚补集三分法（SYN2-R 将改五→四+尾声，三分法不受影响）。
- 无 stepper、无 mermaid。bespoke 一个代码块 scoped `.ch00`。
