# 设计原则（设计系统 v2）—— 排版与可视化宪法

> **为什么有这份文件**（2026-07-04 DSN-01 起；2026-07-08 按"控制机制"重排为 Part 0–3）：
> v1 产出形式主义——固定组件模板套所有章节，导致全章等重盒子海、正文进框、粗体轰炸、流程图灰阶淹没差异点。
> 根因不是缺组件，是缺"什么时候用什么、什么时候不用"的规则，且流水线里没有"设计"这道工序。
>
> **这份文件是一个控制机制**，运行在一个目标不可完全编码、执行者（含作者自己）有已知偏差、验收昂贵且滞后的场景上。
> 它的每一条都是为压低下面四类损失而存在：①规则 ≠ 真目标（规格误差）②执行者被偏差带偏 ③检查成本 ④跨 session 打补丁漂移。
> 所以：**规则尽量管结果不管计数；每条标明是"明线"还是"靠判断"，并带上"它防哪个翻车 / 何时该破例"。**
> tokens 与基础组件仍在 `findings/_design-system.html`（单一真相）。契约 2/3/7 的排版条款以本文件为准；冲突时改契约对齐本文件。

**怎么读这份文件**：
- **Part 0**（一页纸原则）——读一次内化，其余都是它的推论。
- **Part 1**（规则表）——执行时和验收时都查这里；**每条规则只住这一处**，设计门不再复述。
- **Part 2**（每章工序 + 设计门）——每章开工查一次。
- **Part 3**（附录：手法库 + 先例）——**查阅区，不线性读**；写视觉脚本拿不准图式时才翻。

## Part 0 · 一页纸原则（读一次内化）

1. **形式跟随论点。** 判断任一处投入多少，只问一句：**不精心画，读者会不会误解或卡壳？** 会 → 重头，配得上定制；不会 → 从简甚至纯文字。简单的东西简单画，是尊重读者。
2. **要守的是"梯度"，不是"数量"。** v1 的病是**均匀**——什么都强调=什么都没强调。梯度靠**大部分版面安静**挣来，不靠限制响的那几个有几个。所以别数个数，去保证"有明确的第 1 名 + 安静的基线"。
3. **判断不可外包给规则。** 不可编码的核心，规则的职责是**逼执行者把判断做好**（强制排序、论证第 1 名），不是替它拍板（`≤2` 这种）。禁止式规则只留给"执行者在此点判断不可信"的地方（见 Part 1 的"明线"）。
4. **颜色 = 内容坐标。** 全局色板贯穿全书；页内只让颜色编码"读者此刻该分辨的那一个维度"，其余一律灰。差异靠高亮不靠换画法。
5. **裸文优先。** 叙事散文直接呼吸在页面背景上。盒子只留给三类：图、真实制品、判词 callout。
6. **每个抽象主张配一件真实制品。** 真命令 / 真数字 / 真文案原文把主张钉死（与契约 3"必须含真实制品"同源）。

> 每条原则的落地规则在 Part 1，可复用手法与锚例在 Part 3。

## Part 1 · 规则表（执行 + 验收都查这里，每条只住一处）

**判断规则**带四列：`规则 | 防哪个翻车 | 类型 | 过火/破例信号`。
- **类型 = 明线**：执行者在此点判断不可信（作者已知偏差），硬卡，不许自我豁免。
- **类型 = 判断**：上下文多变、执行者可信，给标准让它按章判。
**机械规则**照做即可，不占判断带宽，列成短条。

### 1A 摆每个块：定档与形式

先给每处内容定 轻/中/重 档，再选形式：

| 档 | 适用 | 形式 |
|---|---|---|
| **轻** | 一句话能说清的差异/事实 | 行内对照（`CC：… / Hermes：…` 带色点）、裸文一句、左色规引文 |
| **中** | 结构性对比、多维对照 | `.matrix` 三列表（默认）、`.compare` 双列卡（仅"两边哲学没法逐维对齐"时）、灰阶极简小图 |
| **重** | "不画就看不懂"的核心机制（读者卡壳点） | 定制可视化（语义配色主图 / stepper / 泳道 / 时间线 / 手工排版），允许 bespoke scoped 样式 |

| 规则 | 防哪个翻车 | 类型 | 过火/破例信号 |
|---|---|---|---|
| **梯度而非数量**：不卡各档个数；改为——大部分块必须轻档/裸文；视觉脚本须给全章所有块排一个**全序**、论证第 1 名（最大卡壳点）配得上重档、并指出哪些块构成"安静的基线"。门查排名与基线，不数个数。 | 均匀用力（v1 主病）；也防旧 `≤2/≤1` 硬卡把真需精讲的机制误压低档 | 判断 | 若你在"为了让某档显得克制"而把一个真需要精讲的机制降档——那是过火。梯度靠基线安静挣来，不靠砍分子。 |
| **降档优先**：拿不准放哪档，放低的那档。宁可朴素，不要平均。 | 执行者（作者）向上夸大的偏差 | 明线 | 一章从头到尾每块视觉重量相同 = 没重点，打回。 |
| **裸文 vs 盒子**：叙事散文不进盒子；轻强调用左侧 2px 色规。盒子只给三类：图、真实制品、判词 callout。 | 正文进框、全章盒子海 | 明线 | 出现"说明性散文被四边描边框住"即违规。 |
| **粗体每段 ≤2 处**，只标"读者扫读必撞见的那个词眼"，不标动词串。两家着色（`.cc`/`.hm`）不算配额。 | 粗体轰炸（作者笔癖） | 明线 | 某段真有第 3 个词该强调 → 说明这段在塞两件事，拆段，不是加粗。 |
| **标题必须是论断或问题**，不是栏目名/组件名。「Hero 机制·分步演示」打回；「Hermes 为什么不信模型的"我做完了"」合格。 | 组件名当标题、模板槽位绑架论证 | 明线 | — |
| **标题/立论用专业术语，不用俏皮比喻**：受众是 harness 产品专家，不是小白。标题、副标、thesis、每节论断、verdict 一律用精确的专业名（评测/效果评估/信用分配/控制回路/闭环等），**不裹进「眼/手/尺子/机器」这类故作通俗的比喻**。通俗解释放正文；类比只留给"真的难懂、非类比说不清"的概念，且几乎不进标题。**禁夸张**（「0.5 秒就懂」「一眼看穿」→ 改「这张图的读法很直接：…」）。 | 把专家当小白哄、俏皮话既不精确又轻浮（比黑话更糟）、夸张失真 | 明线 | 标题里出现比喻性名词（非领域术语）、或"X 秒就/一眼就"这类夸张 = 打回，认真重命名。 |
| **重档的义务**：每个重档必须承载视觉脚本认定的卡壳点，且含**真实制品或语义配色的差异高亮**；多个重档时逐个点名各自卡壳点、证明非凑数。特权：通栏宽度、前后大留白、bespoke。 | 重档沦为装饰、凑数 | 判断 | 说不出它解决哪个卡壳点 → 降档。 |

机械规则：
- **列宽**：全书统一一条版心（`--maxw` 1080），**一章内所有块共用同一左右边缘**（正文、图、矩阵、制品全对齐同一右缘）；**禁窄列/破栏混排**。可读性靠短段（渲染后 ≤4 行）、1.8 行距、段间留白，不靠缩短列宽。
- **一段一事**：一段只推进一个论点，超 4 行（渲染后）必拆；说明文字与引文/制品分离，不粘连。
- **分节标题**采编辑式：小字 kicker（栏目定位）+ 大标题（论断本身）。

### 1B 画图与可视化

| 规则 | 防哪个翻车 | 类型 | 过火/破例信号 |
|---|---|---|---|
| **两图对比必须打亮差异节点**：并排两图的唯一理由是让读者 0.5 秒看到差异，差异节点用强调色+粗描边（`classDef`），其余安静。 | 灰阶淹没差异点（v1 病） | 明线 | 自检：遮住标题 3 秒内指不出两图差异 = 白画，返工。 |
| **同一章两侧用同一套图形词汇**，差异靠高亮不靠换画法。 | 两边画法一变、读者没法对齐比较 | 明线 | — |
| **图式选型**：先判"这一章最难的是哪类"（过程/时序/关系/映射/档位/容量/隔离/分层/装配/范式/谱系），再挑对应图式；难点在工程实现就放弃大图走真实制品；一句话能说清就别上重型。选型判据见 Part 3 §3A。 | 硬套 mermaid、为画而画 | 判断 | — |
| **词条可解码**：重档图里高度概括的短语级词条（如「tick 保活」），逐个自问"不看正文能懂吗"——是黑话的必须配 hover 一句话（`data-tip`）或图下附注，图注注明"悬停可见解释"。（判断哪个词算黑话需判断，故非机械。） | 压缩出的黑话没解码通道、读者卡在术语上 | 判断 | 给本就是常识的词也硬加 tip = 噪音；或加了 tip 却没在图注标"悬停可见" |

机械规则：
- 流程图一律 `flowchart TD`（纵向）、短标签、主干精简、边缘分支移到正文。
- **语义配色（强制）**：判断菱形=amber 描边、终点=green 描边、普通动作=中性。
- 图内字号 ≥15px。
- mermaid 非唯一：机制形状是"谱系/脊椎/链条/对照"而非"流程"时，手工 HTML 排版往往更准（Part 3 先例）。
- **合订本离线守卫**：所有 mermaid 调用（含弹窗内）裹 `MERMAID_OK`，否则 CDN 掉线整页白屏。

### 1C Hero 与开篇

Hero 可以是固定配方（learn-cc 证明零变体反降导航成本），但**每个字段必须是硬信息**：
- 章号徽标 + 轴分组色标签 + **论断式 H1** + 一句副标（金句/祈使对偶句，如「挂在旁边，别写进去」）。
- **一行硬指标**（可选推荐）：本章最硬的 2–3 个数字/事实（如「CC 5 种权限模式 · Hermes 默认拒绝」）。
- **核心判断的载体 = thesis 框**（绿左规判词框，判词本是允许进框的三类之一），框内必须分段、一段一事、≤2 段。**裸大字 lede 已废**（与正文连排会挤成一坨、字号层次乱，PM 2026-07-06 裁定）。
- **开篇 story 虚线框**允许：thesis 框 + story 框构成开篇两个视觉锚点；story 框内用 `<p>` 分段（每段 ≤4 行），禁 `<br><br>` 连排长文。**此为开篇专属例外**，正文其余叙事仍裸文优先。
- **砍掉**装饰性关键词 chips 排（视觉噪音 > 信息量）。

### 1D 详情入口与弹窗（给"想追溯细节的读者"留深入口）

| 规则 | 防哪个翻车 | 类型 | 过火/破例信号 |
|---|---|---|---|
| **准入：论点必主线，只有"收据"才准进入口/弹窗。** 自检：把入口后面的内容永久删掉，读者带走那句话会不会变？会变 = 论点，拉回主线（藏起来 = 对略读读者不存在 = 以偏概全，撞忠实度门）；不变 = 收据，可入口。 | 把论点藏进点击后面 | 明线 | — |
| **反诱饵**：入口文案不许把"本节正努力弱化的差别"当钩子吊出来（CH-A1R-02：把「…9 vs 13 输出结构」改成中性的「看两家 prompt 逐字原文」）。 | 文案与本节论点反着劲 | 判断 | 文案在强化一个本节判为"非分野"的差别 → 改中性。 |

机械规则（形态·摆位·权重，锚例见 Part 3）：
- **形态跟随载荷**：集合型入口（露更多同类项）= 集合末尾第 N+1 格、继承集合语汇（`.more`）；证据型入口（露某步完整证据）= 紧贴它所解释的块的一条档三文字链（`↳ 描述 →`）。
- **摆位锚定**：贴着主体（相邻即锚定）优先；必须放远或同处多条时才用脚注式打标（`*`/`ⓘ`）。反面：既不贴主体又不打标 = 孤儿。
- **权重档三·弱**：文字链或幽灵 chip，**不用带框按钮**（带框 = 档二，与真内容抢眼）。
- **弹窗 anatomy**：触发器标签 = 弹窗标题；标题+关闭×、点遮罩/Esc 关、深色主题、内部滚动；内含图受 `MERMAID_OK` 约束。

### 1E 样式归属与 tokens

- **色彩语义（不变）**：`--blue`=Claude Code、`--purple`=Hermes、`--green`=结论/正向、`--amber`=判断/警示、`--red`=危险。背景三阶 `--bg`/`--surface`/`--surface-2`；文字三阶 `--text`/`--muted`/`--faint`。
- **不引入新品牌色**：bespoke 样式只准引用上述 tokens 派生（明线，防 8 章漂移）。
- **两层制**：会被 ≥2 章复用的样式 → 集中层 `findings/_design-system.html`（单一真相，别往 `build.py` 拷第二份）；为一个论点定制、无复用价值 → **章内一个 `<style class="bespoke">` 代码块**（scoped 在本章根 class 下如 `.syn2`；块数 1 个，**不限其内可视化个数**——旧「每章 ≤1 处」指代码块、非可视化个数，勿误读）。`build.py` 的 `extract_bespoke_css` 收进合订本 head。bespoke 里长出复用价值的手法，由后续章节提炼进集中层，禁止复制第二份。

## Part 2 · 每章工序：视觉脚本 + 设计门

### 2.1 视觉脚本（渲染之前的"设计"这道工）

**谁做**：主 session（判断密集，与"综合"同级，不外包渲染员）。**何时**：综合论点收敛后、派渲染员前。**产物**：写进渲染员任务提示的一段脚本，回答五问：

1. **一句话论点**：本章读者只能带走一句话，是哪句？
2. **最大卡壳点**：哪个概念不精心画，PM 读者会误解或读不下去？——这决定重档给谁。
3. **叙事顺序**：问题→机制→差异→判词是默认，但**允许每章不同**（综合②是场景→主图→根因→收纳）。按本章论证逻辑排，不按模板槽位排。
4. **全序重量分配表**：把本章每个论点/机制排成一个**全序**并各标 轻/中/重；顶端论证"第 1 名配得上重档"，底端点名"安静的基线"是哪些块（这一步替代旧的数字配额——见 Part 0 原则 2/3）。
5. **重档草案**：画什么、高亮什么、读者应 0.5 秒看到什么、用哪件真实制品。有思路后查 Part 3 §3A 找参照（借鉴手法，不临摹样式）。

**警示（与契约 7.4 同源）**：视觉减负最易顺手丢机制——"不好画"是换形式的理由，不是降档更不是删除的理由。**全序重量分配表必须覆盖研究 JSON 里的全部机制，谁都不许静默消失。**

### 2.2 设计门（第四道门，与三道内容门并列）

渲染完成后、build 之前，PM 视角逐条过。**门的主体 = 逐条走 Part 1 每条规则、对着它的『过火/破例信号』列查这一章有没有踩**（不复述规则，但信号列就是验收动作，防"我扫了一眼觉得 OK"式敷衍）。加下面四项：

- [ ] **10 秒重点测试**：快速滚一遍，能说出"本章最重要的一张图/一个论断"吗？说不出 = 没有视觉重点，打回。
- [ ] **梯度自查**：视觉脚本给出了全序排名、论证了第 1 名、点名了安静基线；**安静基线（轻档/裸文）须占全章大部分块**（这是"大部分块必须轻档/裸文"规则的可查版——防中档平均用力：一章 5 个 matrix 也能过"有第 1 名"，但过不了这条）；重档逐个过了『不画就看不懂』并点名卡壳点。
- [ ] **抽查（防全表沦为橡皮图章）**：任抽 3 段查粗体 ≤2；任抽 3 个 h2/h3 查是论断非栏目名；两图对比遮住标题 3 秒内能否指出差异。抽中即违规 = 打回，不放行。
- [ ] **完整性挂钩**：对照全序重量分配表，确认没有机制在改版/排版中被静默丢弃（与忠实度门衔接）。
- [ ] 机械两关照旧（契约 2：抽内联脚本 `node --check` + 浏览器 smoke 真点交互 + 图为深色）。

## Part 3 · 附录（查阅区，不线性读）

> 逐页明细见收割存档 `assets/harvest-learn-cc/`（s01–s20 + 版本/架构，22 页）。本节是提炼后的可复用原则。写视觉脚本拿不准图式时才翻。

### 3A 手法库（收割自 learn-claude-code；借鉴手法、不临摹样式）

**用色纪律（最值得学的一条）**：全局固定色板贯穿始终；页内颜色只编码"读者此刻该分辨的那一维"，其余灰（对照站 s01 整图压灰、只留一条蓝标"此刻活跃"；s03 用绿/黄/红直接承载 allow/ask/deny——色即义、无需图例）。概念自带档位/状态时让颜色去承载它。

**重型可视化选型（图式与难点一一对应）**：
- 过程/状态随时间变 → 可 step 的分步件（图+状态面板联动，步进只点亮当前节点）。
- 时序/并发（谁先谁后、谁等谁）→ 带时间轴的多泳道图。
- 关系/依赖（谁挂着谁）→ 拓扑/DAG。
- 一对多映射（分发/扇出）→ 中心节点+放射分支+每支一句白话。
- 档位路由（什么落哪档）→ 泳道流水线+颜色编码档位+真实样本卡。
- 容量/占用（不可见资源）→ 拟物量表（量杯/进度条）+分色图例。
- 隔离/派生（主线 vs 支线）→ 双泳道、末端一根回传箭头；空间隔离 → 分叉文件树。
- 分层存留（哪层丢/留）→ 上下分层图，一层蒸发一层沉底。
- 动态装配（多源拼成品）→ 分色积木拼装图。
- 范式转变（两种世界观）→ 双模式并置心智图。

交互原语**收敛到一种**（组件化 stepper），到处复用、不造新控件。**可点性靠语境不靠边框**：`.more` 虚线 chip 明确读作可点，靠"排集合末尾+箭头+『查看全部 N 种』计数"三个语境信号——修可点性该加语境信号（位置/箭头/计数/hover），不是加边框重量。

**对比表达（版本对比页专记，与全书同构）**：
- 对比原子 = 三列表 `维度 | Claude Code | Hermes`（`.matrix` 骨架，对比默认形式）——先立维度、再同行并置，差异靠"横扫同一行"获得。
- 差异高亮克制：同行并置本身就是差异表达，不满屏红绿；仅"一方有/一方无"的质变才用一个语义色徽章点一下。
- `.compare` 双列卡是**弱形式**（两坨散文并排、读者自己找对应点），只用于"两边哲学没法逐维对齐"；能逐维对齐一律 `.matrix`。
- **卡内只放标签级短语（一两行），解释放到卡外的裸文短段（渲染后 ≤4 行）**——卡片是并列对比的骨架，不是装散文的盒子（卡内塞散文＝v1 盒子海的近亲）。
- 密度延迟释放：维度多时先分组/聚焦，读者选中再展开。三列表讲逐维差异（微观），定位矩阵/色带讲全局格局（宏观），别混用。

**成对概念共用隐喻**：相邻/对立概念用同一套图形语言（s08 压缩 ↔ s09 记忆共用"层/容量"隐喻），让差异落在同一坐标系里可比。→ CC/Hermes 天然成对，同一章两侧必须同一套图形词汇。

**文案与证据**：副标/金句用祈使对偶句一句立住主张（机制越简单副标越该短到像口号）；"反例开场"（没有 X 会怎么翻车）比正面说教更快让读者认同；实现细节压进页脚/折叠区，正文与图只讲"为什么/是什么"。

**骨架的一致与弹性（一处张力，诚实标注）**：对照站锁死七段式模板靠重复降导航成本，与我们"骨架降为默认可偏离"方向相反。两边各取一半——**段落功能层面保持一致**（读者学一次就知道"判断在头、深潜在中、诚实边界在尾"，骨架仍是强默认，预计 10 章里 8 章照走）；**偏离必须有论证理由**且写进视觉脚本。一致性服务导航，弹性服务论证，冲突时论证优先。
