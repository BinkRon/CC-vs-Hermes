## 0. 什么是 harness（本质定义）

> harness = 把一个**无状态的 next-token 函数**，包装成**有状态、能行动、会自己决定何时停、并能跨时间存在**的 agent 的那整层脚手架。

它不是模型本身，也不是某个具体工具的实现，而是介于两者之间的控制结构。

---

## 1. 框架：两条正交轴（WHAT）+ 一个映射（WHY / WHERE）

### 轴一 · Loop 之内（一个循环的解剖）
> 演绎而来（从"模型是无状态函数、必须被循环"推出）。**完备但浅，不随模型变。**

| 侧 | 职责 |
|---|---|
| **输入侧** | 组装 context——决定这一次调用窗口里放什么 |
| **计算侧** | 调模型，拿输出 |
| **输出侧** | 解析输出（tool call？文本？停止信号？）→ 在某个**环境**上执行动作 → 捕获结果 |
| **控制侧** | 决定是否再循环 / 何时停 / 持久化状态 / 强制约束（迭代上限、超时、权限、护栏）/ 验证 |

```
while 没结束:                # 控制侧：循环本身 + 何时停
    ctx = 组装上下文()        # 输入侧
    out = 调模型(ctx)         # 计算侧
    结果 = 执行(out)          # 输出侧
    判断是否继续(结果)         # 控制侧
```

局限：这条轴保证"不漏一整个环节"，但很浅（不告诉你每侧内部有什么），且默认了"单轮、同步、有人在场"。

### 轴二 · Loop 之外（loop 与其补集的五种关系）
> 起初归纳而来（轴一公理默默丢掉的部分，由"功能目录审计"逼出，得四格）。**2026-07-03 升级为演绎结构**："Loop 之外" = loop 的**补集**，补集可穷尽划分——不是 loop 的东西，要么是**非主体**（时空，再切为"何时存在 / 作用何地"），要么是**其他主体**（对它有权威的人 / 平级的同类 loop），要么是**它自己的过去与未来**。得五种关系，完备性来自补集划分——轴二自此与轴一同等地位（演绎保完备）。
>
> 方法论注（元规则 1 的实测闭环）：归纳先摸到 4/5；第五格（对自己=学习闭环）是被 **Hermes 自家 README/官网**（"The self-improving AI agent"）当审计员照出来的——当初 CC 形状的功能目录里没有它。演绎补全后还预言了一个盲点：「对同类」应含**外来 agent 互通**（如 Hermes `acp_adapter`，07 未覆盖，待查证）。
> **复审点（防"干净弧线"免检）**：五关系的完备性与"4/5"这个自评，在 **06、08 完成且 ACP 查证落地后复审一次**再定稿——若第五格材料撑不起、或又冒出第六类关系，回来改本节，不许因为故事顺就盖章。

| 关系 | 回答什么 | 章 |
|---|---|---|
| **① 对时间** | 谁/何时启动循环；谁保活；同步阻塞 vs 后台并发；有人启动 vs 定时/事件 | 04 |
| **② 对人（用户）** | 人如何支配它——**事前编程**（hooks/skills/MCP/plugins → 05 章）＋**在场治理**（注意力/监督/拍板 → 不立章，作综合②脊椎） | 05 ＋ 综合② |

| **⑤ 对自己** | 跨实例什么被保留、系统如何变强：记忆蒸馏 / 技能自造与自我改进 / 用户建模 / 轨迹→训练 | 08（新增） |

### 映射 · A 缺陷补偿地图（WHY 每块存在 / WHERE 该修）
> 不是一条"划分轴"，而是**架在划分之上的映射**：给每个部件标上成因（补哪个模型缺陷），并回答"修它的地方在模型还是在壳"。

裸模型的缺陷 → harness 用哪块补：

| 模型缺陷 | 补偿部件 |
|---|---|
| 无状态、只算一次 | 控制循环 |
| 上下文窗口有限 | 上下文 / 记忆管理 |
| 碰不到真实世界 | 工具系统 + 环境 |
| 不可靠、不对齐 | 安全 / 权限 + 验证 |
| 单窗口、单视角窄 | 多智能体 |
| **权重冻结、不从经验变强** | **学习闭环**（壳内：记忆蒸馏/技能自造/用户模型；壳外：轨迹→再训练）——2026-07-03 补 |

**A 的适用域边界（关键结论）：**
- **对轴一（loop 之内）完美适用**——每块内层部件都对应一个模型缺陷。这里主问法成立：**「改模型 vs 改壳」**。
- **对轴二（loop 之外）失效**——那些部件补的不是模型缺陷，而是**世界/人/部署现实**：
  - 可扩展性/Hooks 补的是"壳设计者预料不到所有用法"（人的局限）；
  - 环境/substrate 补的是"世界本身危险且共享"（与模型无关）；
  - 时间/触发 一半是模型"纯被动、单发"，一半是部署需求。
- **∴ A 的失效点 = harness 从"面向模型"转向"面向世界"的分水岭。**
- **例外：⑤ 对自己（学习闭环）坐在两把仪器的接缝上**——它既有轴二问法（什么现实逼出它：定位"grows with you" + 模型作者的数据飞轮），又让轴一问法复活（学习进**权重**还是进**壳**：轨迹→训练 vs 记忆/技能=不用梯度下降的学习）。它是两轴物理相连的那个格子（壳学习产生的轨迹正是权重学习的原料）。

## 2. 对比的头号假设（贯穿始终）

> **Nous 是模型作者，Anthropic（对 Claude Code 而言）只做壳。**

Hermes 由 NousResearch 自训模型 → 计算侧不是黑盒，可**模型-壳联合设计**（把 tool-call 格式、trajectory 格式直接训进模型）。Claude Code 把模型当**固定契约**，只能在壳里补。

推论 —— 两种问法按轴切换：
- **Loop 之内**：追问「这家把它补在壳里，还是训进模型里？」
- **Loop 之外**：追问「是什么世界/人/部署现实逼出了它？」

---

## 3. 两条方法论元规则

1. **演绎 vs 归纳，缺一不可**：完备来自演绎（轴一），深度/显著性来自归纳。二者不可兼得于一个框架，故合用。
2. **功能目录当审计员**：不优雅但完整的功能清单（如某产品的全部 feature），因不自我审查，能照出原则框架的公理默默漏掉的东西。但**别拿它当中立对比尺**——它带产品偏见（围着某一家的功能名长出来），会诱导你"在 B 里找 A 的功能，找不到就判缺失"。

## 4. 调研计划

**划分依据 = 框架（保证不漏、每篇对应一个干净的格子）；排序与深度 = 优先级（哪篇先跑、投多少 effort）。二者分开，不再混用。**

**大顺序：先啃完整条轴一（Loop 之内）→ 收成一个故事 → 再啃整条轴二（Loop 之外）→ 收成一个故事。** 每轴内部按优先级排序。目标不是攒齐 N 篇，而是最终能回答两个问题：**「一个 Loop 是怎么设计的」**和**「Loop 之外是怎么设计的」**。
