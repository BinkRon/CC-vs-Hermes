第 0 篇 · 导读拆法与读法

导读：比的是壳，不是模型

两条轴、两把仪器、十章正文——读任何一章之前，先拿到这张地图。

TL;DR

这本册子对比 Claude Code 与 Hermes Agent 的 harness——包在模型外面、把它变成能干活的 agent 的那整层控制脚手架。两家产品的表面能力高度相似，值得比的差异几乎都住在壳这一层；而对比有一个贯穿全书的支点：Hermes 的作者自己训练模型，Claude Code 的壳团队把模型当固定契约。

拆法是两条正交轴：轴一解剖一个回合之内（输入 / 计算 / 输出 / 控制四侧），轴二穷尽回合之外（时空、其他主体、它自己）。每章带一把固定的提问仪器。这篇导读只给地图与问法，不给任何一章的结论。

00问题：两个都能干活的 agent，怎么比才不漏、也不偏

把两个 agent 放在一起比，有两种省力的写法。第一种，拿 A 家的功能清单去 B 家里逐项找，找不到就记一笔「缺失」——清单是围着 A 的产品形状长出来的，这么比从第一行起就带着偏见。第二种，各写各的使用体感——读起来热闹，但回答不了一个基本问题：有没有整块维度被漏掉？体感不自带完备性。

所以这项研究先不急着比，先回答一个更基础的问题：一个 agent 的 harness，总共由哪些「格子」构成？这个划分必须不依附任何一家的产品形态，还得说清自己凭什么穷尽。然后，才把两家的做法填进格子里。这本册子的两条轴，就是那次划分的结果。

01对象与支点：一家自己训模型，一家把模型当固定契约

被比较的两家：Claude Code 是 Anthropic 的编码 agent，驱动它的模型是 Claude；Hermes Agent 出自 NousResearch，驱动它的模型是自家训练的 Hermes 系列。两个都是「模型＋壳」的复合体，这本册子只比壳。

壳，就是 harness：模型本身是个一次性函数——没有记性，碰不到真实世界，也不知道什么时候该停；把它包装成有状态、能行动、会自己决定何时停、能跨时间存在的 agent 的那整层脚手架，就是 harness。表面上两家都能听懂任务、调工具、改文件，但「怎么做到」的分歧，几乎全部住在这一层。

全书的对比有一个支点。它不是研究结论，是公开事实：Hermes 的作者训练自己的模型，壳想要的行为可以直接训进权重；对 Claude Code 的壳团队而言，模型是一份固定契约，一切只能在壳里想办法。同一个问题摆在两家面前，一家永远比另一家多一个选项——这个不对称会在全书反复出现。

内圈（实线）：从「模型必须被循环调用」演绎出的四侧解剖。外圈（虚线）：按补集切出的回合之外——章号即左侧栏目录。

轴一从一个绕不开的事实推出来：模型一次只算一步，必须被外面的循环反复调用，才成为一个能把事做完的 agent。于是一个回合之内必然有四件事——拼材料、算一步、执行动作、决定去留。这条轴是演绎出来的，保证不漏一整个环节；代价是浅——它不告诉你每一侧内部有什么，还默默假设了「单轮、同步、有人在场」。

轴二负责把这个假设丢掉的东西捡回来。定义很朴素：不属于循环本身的一切。而「不属于循环的」可以按补集分干净：它所处的时空（何时被唤起、动作落在何地）、它之外的其他主体（对它有权威的人、与它平级的同类）、以及它自己的过去与未来。三类摊开，就是轴二的各章。敢说「不漏」，凭的是这个划分方式，不是列举的勤奋。

这套切法上过一次现实的审计。研究早期靠归纳只摸到「时空」与「其他主体」名下的四章；第五章——loop 与它自己——是被 Hermes 官方仓库自述的第一句话照出来的：

「The self-improving AI agent built by Nous Research.」
Hermes Agent · 仓库 README 首句

当时手里那份按另一家产品形状长出来的功能清单里，没有这一格。这也定下了全书对功能清单的用法：不当尺子，只当审计员——清单不自我审查，恰好能照出优雅框架默默漏掉的东西。

03读法：每章带一把固定的提问仪器

轴一的三章共用一个问法：「这个行为，是训进模型里的，还是写进壳里的？」同一个可靠性要求，一家可以压进权重，一家只能靠壳里的工程。看两家各把它放在哪，比罗列功能表深一层——这也是「计算侧不立章」的原因：它的材料就散在轴一各章的这个问法里。

轴二换一个问法：「什么现实逼出了它？」回合之外的部件，补的往往不再是模型的缺陷，而是世界、人和部署环境的现实——「训模型还是改壳」在这里大多失去意义，仪器跟着换。

唯一的例外是 08 章：self-improvement 坐在两条轴的接缝上，两把仪器都用——既问什么现实逼出了它，也问学到的东西进了权重还是进了壳。每章末尾的 verdict 由当章仪器给出；这篇导读不预演其中任何一个。

05方法与边界：每个论断都能被追问出处

全书的事实性论断都对着两家的真实代码核验过，出处以文件与行号的形式存在各章页脚；正文不出现代码，但真实的提示词原文、错误消息与阈值数字会作为证据直接展示。每篇成稿要过独立的质量门才收录：作者自审、对照真实代码的事实核验、独立的完整性核验（专防「为了讲一个干净的故事而丢掉机制」），以及一道排版设计门。

两条边界请读者带着。其一，版本切片：两家都在快速迭代，各章描述的是核验时刻的代码事实——研究期间 Hermes 主线明显前移，轴二各章都在章末注明了自己核验的切片。其二，默认不等于绝对：凡写「默认行为」的地方不外推成「永远如此」，实验性、未对普通用户开放的能力会被明确标注——这些交代通常收在各章末尾的边界说明里。
