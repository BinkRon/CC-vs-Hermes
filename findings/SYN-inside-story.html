一个 Loop 是怎么设计的

三侧看完，两家的 Loop 其实只由一个变量决定——你有多信任模型。

信任决定壳的厚度 单杠杆 vs 双杠杆 改模型 vs 改壳 模型作者反而更不信任
控制侧、输入侧、输出侧看似三件事，其实是同一个选择的三次显形——这个 harness 有多信任模型，就决定了壳做多薄。Claude Code 押注"一个可控、持续变强的托管模型"，于是三侧都选择信任模型、把壳做薄；Hermes 面对"自训 + 第三方、良莠不齐的开源模型群"，于是三侧都选择不全信、用壳（有时连同训练）加厚兜底。

01两句话，两种哲学

Claude Code

信任模型，壳做薄。能交给模型判断的就交给它（何时停、怎么总结、怎么表达意图），壳只做管道——执行、翻译、守资源。好处：能力天花板随模型升级自动抬高。代价：模型犯浑（跑偏、做错）时，薄壳基本不纠正——护栏只挡「烧钱 / 闯祸」，不挡「做得对不对」。

Hermes Agent

谁都不全信，模型和壳一起加固。既然自己能训模型，就把一部分可靠性训进权重；又因为要跑参差不齐的开源/第三方模型，还得在壳里加一层防线。两根杠杆一起用。代价：壳复杂、要持续维护。

02同一个哲学，在三侧的三次显形

控制、输入、输出行动——三个不同的技术问题，暴露出 Hermes 的两根杠杆：控制侧、输出侧是信任驱动的"改壳"（越不敢信模型，壳越厚）；输入侧是所有权驱动的"改模型"（拥有模型，才能把能力训进权重）。Claude Code 只有前一根、还基本不用。

2.1控制侧 · 何时算做完

Claude Code

默认模型说了算——这轮没点工具＝完成。壳默认不判断"活干完没"（停止钩子、token 预算续跑是两个 opt-in 覆盖口，默认不用）。

Hermes Agent

壳设一道 verify-before-done 门：模型改了代码没测试证据就被推回循环，无权单方面宣布做完。（此门 surface-aware：编码/程序化入口默认开、聊天平台默认关，且只对代码改动触发。）

判词——信任问题，动的是壳杠杆：CC 默认信任模型的自述→壳默认不设门（但留了停止钩子/预算续跑两个 opt-in 口）；Hermes 默认不信任任何模型（含自家）的"我做完了"→在编码入口加验证门（聊天入口默认关）。（改壳）

2.2输入侧 · 上下文满了谁来压

Claude Code

把压缩的智能外包给通用模型（壳只给一段提示词让它自己写摘要）。

Hermes Agent

把压缩形态既训进模型、又写进壳（同源管线造训练数据，模型见过被压的历史）。

判词——这一侧不是"信不信"，是"谁拥有模型"，动的是模型杠杆：CC 不拥有模型，只能给通用模型写提示、临场发挥（改壳）；Hermes 拥有模型，用同源管线造"压缩过"的训练数据、让模型见过被摘要的历史（意图把这形态训进权重——"已训进线上权重"非本仓库可证，但这条改模型的路只有模型作者走得通）。这正是 Hermes 多出的第二根杠杆。

2.3输出行动侧 · 怎么解析意图、结果怎么回

Claude Code

读模型原生结构化格式，壳几乎不设防。

Hermes Agent

运行时同样读结构化，但壳额外加一层可靠性纠偏（工具名修复 / 空名防"鹦鹉学舌" / 剥掉正文里的假工具调用），以兜住弱模型。

判词——信任问题，动的是壳杠杆：CC 信任那一个托管模型的格式→壳不设防；Hermes 不敢假设第三方弱模型可靠→壳加纠偏层。（改壳）

03反直觉：做模型的人，壳反而做得更厚

直觉会以为：Nous 自己训模型，应该最信任它、壳做得最薄。实际恰恰相反——它的壳更厚。关键是看清它不信任的到底是谁：

  • 对任何模型（含自家）的"我做完了"自述——不信，所以有 verify 门（这一层对自家模型也设防；不过它 surface-aware——编码入口默认开、聊天平台默认关）。
  • 对第三方开源弱模型的格式可靠性——不敢假设，所以有纠偏层（这一层主要不为自家模型）。

根源是处境不同：Anthropic 只发一个自己完全掌控、持续变强的模型，Claude Code 可以放心押它；Nous 活在开源世界——模型不止一个、质量参差、还要自托管——它无法假设可靠，于是两头下注：能训进权重的就训进去（改模型），兜不住的用壳挡（改壳）。

"改模型 vs 改壳"的答案：Claude Code＝单杠杆（信任那一个托管模型）；Hermes＝双杠杆（拥有模型→能改模型；面对异质生态→又必须改壳）。

04对产品经理意味着什么

  • 若你基于一个强托管模型做 agent：学 Claude Code——壳做薄、把判断权交给模型，天花板随模型涨；但要接受"模型犯浑时壳兜不住"。
  • 若你要跑多种/开源/自托管模型：学 Hermes——壳必须厚；如果你能训模型，尽量把可靠性训进去；代价是壳复杂、要持续维护。
  • 一句话记住：壳的厚度不是能力问题，是信任问题。你越不能假设模型可靠，壳就越得厚。
