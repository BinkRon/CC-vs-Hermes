Loop 与自己

越用越强，是真的还是宣传？

self-improving 是 README 第一句 改权重＝谁拥有模型说了算 CC 结构上没有训练代码 Hermes 有管线但不接你对话 两家都证不到权重级变强
「越用越强」这句宣传，落到代码里要先分成两半：一半是改壳（把经验写成记忆文件，越用越懂你——这半是真的，另有『它如何适应你』一章专讲）；另一半是改权重——把你的使用变成模型本身更聪明。本章只钉后一半。而后一半能不能成立，答案又一次落在轴一那根「谁拥有模型」杠杆上：CC 不拥有模型，结构上根本没有「改权重」这条路，仓库里连一行训练代码都没有，只能把它外包给母公司的下一代基座；Hermes 拥有模型，仓库里真有一条产训练数据的管线——但那条管线吃的是研究员策划好的数据集，根本不接你的真实对话。所以一处对称的诚实必须先摆上桌：两家仓库都证不到「你的使用让它权重级变强」——CC 断在「没那个能力」，Hermes 断在「有能力却没把管子接到你身上」。「self-improving / 越用越强」这个飞轮，两家都只证到了前半段，后半段都得标『意图，仓库外不可证』。

01问题：跑过一千次任务，它自己变聪明了吗？

用产品语言讲：「越用越强」有两种截然不同的含义——一种是它记住了你（改壳），一种是它这个模型本身变强了（改权重）。宣传把两者混着讲，本章只钉后一种。

它替你跑了一千次任务、被你纠正过几百回。它这个模型因此变聪明了吗？还是每一次调用的，都是同一个出厂时就定型、你怎么用都不会长进的它？

这不是抬杠。Hermes 的 README 第一句就把自己定义成「self-improving AI agent」「唯一自带学习闭环」，还专列一行讲「产轨迹、压缩，用来训练下一代工具调用模型」。这些词听起来像是「你越用，它这个脑子越聪明」。所以问题很具体：这条从『你的使用』通到『更聪明的权重』的飞轮，落到两个仓库的代码里，到底哪一段是真接通的、哪一段只是意图？

02核心机制：能不能改权重，是「谁拥有模型」说了算

「改权重」不是一个功能开关，而是一个结构性前提：你得先拥有那个模型，才谈得上把使用喂回去重训它。这一条，把两家从起点就劈成两极。

Claude Code · 不拥有模型 → 这条路结构上就不存在

CC 是托管产品，调用的是母公司的模型，不拥有权重。所以它仓库里没有、也不可能有任何训练代码。它能做的极限，是在你逐次同意下，把对话转录上传给母公司当训练素材（仅抹掉密钥、绑你的账户，并非匿名）——这是单向投喂，喂进去之后能不能变成更强的下一代基座、什么时候变，全在仓库之外、CC 自己看不到也证不到。改权重这件事，根本不在它手里。

Hermes · 拥有模型 → 握着杠杆，但没把管子接到你

Hermes 出自自训模型的 Nous，拥有权重，仓库里真备了一条产训练数据的管线：批量跑任务、生成轨迹、压缩保训练信号。这根「把使用变权重」的杠杆，它确实握在手里。但关键在于管线的进料口：那条批量管线吃的是一份研究员策划好的任务数据集，跟你平时聊天攒下的真实对话库之间，代码里没有任何自动接线。它有能力闭合飞轮，却没把入口对准你。

同一根杠杆的两极：「改权重」这条路存不存在，先看你拥不拥有模型。CC 不拥有 → 路本身不存在，只能把权重级变强外包给母公司；Hermes 拥有 → 路存在、管线也真在，但入口没接你的会话。所以两家都到不了「你的使用让它权重级变强」这个终点——一个没有能力，一个有能力没接线。

03深潜 · 把飞轮从中间切开：宣传怎么说，代码怎么接

「self-improving」的飞轮，从「你的使用」到「更聪明的权重」，中间要走好几段。把它从中间切开看，你会发现两家的代码都只接通了前半段，后半段全是仓库外的意图。先看宣传原文，再看代码到底接到了哪里。

宣传怎么说：Hermes README 把「学习闭环 + 训下一代」写进门面

README 用词很重：self-improving、closed learning loop、训练下一代模型。读者很容易把它理解成「我越用，这个模型越聪明」。

README.md · self-improving 定位 + 学习闭环 + 训下一代The self-improving AI agent built by Nous Research. It's the only agent with a built-in learning loop … （A closed learning loop）Agent-curated memory with periodic nudges. Autonomous skill creation after complex tasks. Skills self-improve during use … （Research-ready）Batch trajectory generation, trajectory compression for training the next generation of tool-calling models.

CC 代码怎么接：没有训练代码，只有匿名·单向·仓库外

CC 仓库里搜不到任何训练管线。唯一跟「训练」沾边的，是一处账户级的同意开关和逐次同意的转录上传——上传的是完整原始转录，只抹掉密钥、绑你的账户，并不匿名、也不剥代码/路径（真正剥代码/路径的是另一套「遥测」，那是产品分析、不是训练素材）。连它自己的注释都把训练数据采集定义成「给母公司攒训练数据、别浪费标注员的时间」——也就是说，你的使用是喂给一条你看不见的外部标注/训练流水线的原料，而不是它自己能闭合的循环。

src · 代码注释把上传定义成「攒训练数据、别浪费标注员时间」For training-data collection, a model response conditioned on synthetic placeholders is tainted — fail the trajectory rather than waste labeler time on a turn that will be rejected at submission anyway.

Hermes 代码怎么接：管线吃策划集，轨迹导出默认关

那条批量管线的入参明写着 dataset_file——一份预先备好的 JSONL 任务集，跟你的对话库两回事。而唯一能把当前这次对话存成轨迹的开关，代码里默认是关的、要手动开，存下来也只是本地一个 JSONL 文件，既不自动训练、也不流进那条批量管线。两条管子之间，没有一行代码把它们接起来。

src · 批量管线吃策划数据集（--dataset_file）python batch_runner.py --dataset_file=data.jsonl --batch_size=10 --run_name=my_run dataset_file (str): Path to the dataset JSONL file with 'prompt' field
src · 会话轨迹导出默认关、只写本地 JSONLsave_trajectories: bool = False Save conversation trajectories to JSONL files (appends to trajectory_samples.jsonl). Defaults to False.
飞轮切开后剩下什么：前半段两家都可证——CC 有匿名转录上传，Hermes 有产轨迹的管线。后半段「训进权重、下一代更聪明」两家都证不到、都在仓库外。而最锋利的一刀是给 Hermes 的：就算它拥有模型、管线也真在，那条管线的进料口对准的是策划数据集、不是你的对话——所以连「你每天用它 → 你的使用自动变成训练燃料」这句都不成立。别把「有产数据的能力」讲成「你正在把它喂强」。

06悬而未决 · 防夸大红线（飞轮最容易被讲过头）

  • [两家 · 飞轮从中间切开] 「越用越强」这句话，落到两个仓库里都只证到前半段：CC 证到「经同意匿名上传了转录 / 翻了一个训练同意开关」，Hermes 证到「有一条产训练数据的管线」。后半段「训进权重、下一代更聪明」两家都是意图、都发生在仓库之外，必须对称地标「意图，非本仓库可证」——不许只给一家戴这顶帽子。
  • [Hermes · 最硬的一刀] 就算 Hermes 拥有模型，它那条产训练数据的管线吃的是研究员策划好的任务数据集、不是你的真实对话库——两者在代码里没有自动接线。三条证据：① 批量管线的入参是 dataset_file（预备的 JSONL 任务集）；② 唯一能存当前对话的轨迹开关默认关、要手动开；③ 就算开了也只写本地一个 JSONL 文件，不流进那条批量管线。所以连「你每天用它 → 你的对话自动变成训练燃料」都不成立。别把「有产数据的能力」写成「你的使用正在把它喂强」。
  • [CC · 结构上就没有] CC 仓库里没有一行训练代码，也不可能有——它不拥有模型。它的转录上传是匿名、单向、逐次同意的，喂进的是母公司一条它看不见的外部流水线。别把「上传转录」写成「它在自我训练」——素材汇给别人训下一代，跟「它自己变强」是两回事。
  • [不评优劣 · 时间切片] 本篇只讲「飞轮的形态与可证边界」，不评两条路孰优——取决于你要的是能独立发货的托管产品，还是拥有模型、有潜力闭合飞轮的自托管栈。切片：CC main / Hermes @daf4f1a7a。
  • [另有一章 · 别把两半混讲] 「越用越强」真正落地的那半——它记住你、越用越懂你（改壳·记忆）——不在本章，见「它如何适应你」一章。本章只钉改权重那半飞轮；两半别混成一句「它在自我进化」。
